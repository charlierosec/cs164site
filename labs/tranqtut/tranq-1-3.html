To Assign is to Store

In Tranquility, unlike most mainstream languages, the colon ( : ) is used for assignment. Most mainstream languages however use the equals sign ( = ) for assignment. It is often said that this is like an equality in mathematic, but there are important differences that you need to internalize. In math, if we make a statement such as y =2 x +5, then we are typically establishing a relationship between x and y that remains true thoughout the problem we're solving or proof we're developing. In other words, no matter what x is, y will always be 2 x +5.

Establishing a relationship is not the right way to think about assignment in programming. The reason is that programming is not about establishing static relationships; it's about specifying a sequence of actions through time. So if the current value of x is 3, then when we execute the statement y =2 x +5, then y will get the value 11. That much is what we'd expect from math, but if later x changes, then y doesn't change correspondingly. Similarly, we can't assert that y is a particular value and expect that the computer will solve for x . The way we want to think about this is that the assignment sets y to 11 at a particular point in time, but that later x and y can change independently of each other.
